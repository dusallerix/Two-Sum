# Two-Sum
Задача с [Leetcode](https://leetcode.com/problems/two-sum/description/).

## Описание задачи

>"Given an array of integers nums and an integer target, return indices of the two numbers such that they add up to target.  
You may assume that each input would have exactly one solution, and you may not use the same element twice.  
You can return the answer in any order."

Дан массив целых чисел nums и целое число target. Верните индексы этих двух чисел так, чтобы их сумма равнялась target.  
Предположим, что для каждого входного значения существует ровно одно решение, и нельзя использовать один и тот же элемент дважды.
Ответ можно возвращать в любом порядке.  

Example 1:
|Input:|Output:|Explanation:|
|-|-|-|
|nums = [2,7,11,15] <br>target = 9|[0,1]|nums[0] + nums[1] == 9, we return [0, 1]|

Example 2:
|Input:|Output:|
|-|-|
|nums = [3,2,4] <br>target = 6|[1,2]|

Example 3:

|Input:|Output:|Explanation:|
|-|-|-|
|nums = [3,3] <br>target = 6|[0,1]|Ответ [0,0] будет ошибкой из-за условия|

## Логика решения

Есть два варианта: алгоритмически верный и с indexOf. Оба решения одинаковые по структуре, имеют примерно одинаковую память и время выполнения (33-36ms).   
  
Можно сказать, что они различны только в том случае, что первый вариант зависим от первого числа, подбирая к нему пару, а во втором - пара сама должна найти то первое число.

1) **Два указателя.**  
Простой перебор чисел. Первый указатель ставим на первое число массива и вычитаем из target, находя искомое нам число, второй указатель перебирает остальные числа из массива в цикле. 
Если в течение цикла не получается найти подходящее число, сдвигаем первый указатель на следующий индекс и повторяем перебор.

2) **С помощью indexOf().**  
Решение не совсем верно с той точки, что не во всех языках есть такая функция, но если не нужно придумывать велосипед - то значит можно использовать.  
indexOf() по сути тот же самый второй указатель, который делает АНАЛОГИЧНЫЙ перебор в массиве и возвращает его индекс.
